# Backtest Plan

Historical replay of the full PolyArb pipeline (detect → optimize → simulate) over 30 days of Polymarket data, running in an isolated database so live paper trading is untouched.

## What was added

### New files

- `scripts/backtest_setup.py` — Creates `polyarb_backtest` database, copies Markets + MarketPairs from live DB (read-only)
- `scripts/backfill_history.py` — Fetches 30 days of daily price candles from Polymarket CLOB `/prices-history` endpoint, inserts as PriceSnapshots
- `scripts/backtest.py` — Steps day-by-day through backfilled data, runs full pipeline at each step, outputs summary + JSON report
- `scripts/Dockerfile` — Combines deps from all services so backtest scripts can import detector, optimizer, and simulator code

### Modified files

- `docker-compose.yml` — Added three `backtest` profile services: `backtest-setup`, `backfill`, `backtest`

## Architecture

```
Live DB (polyarb)                    Backtest DB (polyarb_backtest)
┌─────────────────┐   copy (r/o)    ┌─────────────────┐
│ Markets         │ ───────────────► │ Markets         │
│ MarketPairs     │ ───────────────► │ MarketPairs     │
│ PriceSnapshots  │                  │ PriceSnapshots  │ ← backfilled from API
│ PaperTrades     │                  │ PaperTrades     │ ← generated by backtest
│ Portfolio       │                  │ Portfolio       │ ← generated by backtest
└─────────────────┘                  └─────────────────┘
     untouched                            isolated
```

## How to run

```bash
# 1. Create backtest DB + copy detected pairs from live
docker compose --profile backtest run --rm backtest-setup

# 2. Backfill 30 days of historical prices into backtest DB
docker compose --profile backtest run --rm backfill

# 3. Run full pipeline replay
docker compose --profile backtest run --rm backtest
```

Or without Docker (pointing at local Postgres):

```bash
python -m scripts.backtest_setup
POSTGRES_DB=polyarb_backtest python -m scripts.backfill_history --days 30
POSTGRES_DB=polyarb_backtest python -m scripts.backtest --days 30 --capital 10000
```

## Key design decisions

- **Same Postgres instance, separate database** — no extra containers, full isolation
- **`backtest-setup` reads live DB read-only** — copies Markets and MarketPairs, never writes to live
- **`backfill` and `backtest` get `POSTGRES_DB=polyarb_backtest` env override** — all writes go to backtest DB
- **Pair detection is not re-run daily** — semantic similarity + LLM classification finds structural relationships that don't change day-to-day, so existing pairs are reused; optimization + simulation ARE re-run at each daily step with that day's prices
- **Daily granularity** — ~30 time steps, keeps runtime manageable
- **Polymarket CLOB API** — `GET /prices-history?market={token_id}&interval=1d&startTs=...&endTs=...`, no auth required

## Output

The backtest prints an ASCII summary table and writes `backtest_report.json` with:

- Daily portfolio value, cash, realized/unrealized PnL, trade count
- Summary stats: total return %, max drawdown %, annualized Sharpe ratio

## Things to watch for

- The backfiller rate-limits at ~2 req/s to stay under Polymarket's limits — backfilling 200 markets takes ~10-15 minutes
- If you have no MarketPairs yet (detector hasn't run), `backtest-setup` will copy zero pairs and the backtest won't find any opportunities — run the live system first to build up pairs
- `build_constraint_matrix` is imported from `services.detector.constraints` — make sure that module doesn't have side effects or require Redis at import time
